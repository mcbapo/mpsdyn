MPS Code

Matrix Product State Code

This is a basic code which implements the basic functionality needed to do calculations with Matrix Product States, including variational searches for ground state and excited staes of MPOs, time evolution (via Trotterization) and some additional methods. The code structure is described below.

Structure

A brief description of the folder structure is given below:

  • bin
    Contains the (user specific) makefiles and the binary files after compiling. To use the library, each new user should create a file Make.user.USERNAME in this directory (copy from example) and include here their targets, directories and rules.
    Start with "make test1" and run the program "./test1" (which tests functionality of basic tensor class and operations), to check the installation is correct. Run "make clean" to remove all objects from the directory, or "make cleanall" to remove also executables.
  • config
    Contains configuration files which are used as input to set the parameters of a run.
  • core
    Contains all header and cpp files which provide the core functionality such as MPS, basic Hamiltonians and more general MPOs, Tensor contractions and other things.
  • libs
    Folder to store the libraries needed on to link against for certain operating systems. See below for requirements.
  • models
    A collection of various models which have been developed by the users and have proven to be reliable over the time.
  • projects
    Contains a collection of projects which are currently developed by the users or in collaboration among several users. As a new user, you should add a directory for your own here.
  • programs
    Contains a collection of various programs. Also separated by username.
  • tests
    In this folder a collection of tests is stored which are there to test the basic functions and a few basic models of the MPS code. Compiling these tests is also a useful starting point for new users to see if everything is set up properly on their machine and is working.
  • wrapper
    A bunch of wrappers to communicate with blas and the various eigensolvers employed in the code.

Requirements

To be able to compile and run the code one needs a BLAS implementation, such as LAPACK, ARPACK and the PRIMME libraries for the eigensolver. The PRIMME libraries can be downloaded here.

Linux

After compiling the (static) PRIMME libraries libdprimme.a, libprimme.a and libzprimme.a, they have to be put in /usr/local/lib to be compliant with the Makefile structure
