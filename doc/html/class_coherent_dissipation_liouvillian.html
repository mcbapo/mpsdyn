Detailed Description

Program coherent: Compute the steady state of a certain Liouvillian as in with g, gamma given as input parameters.

To compile: make coherent

Receives arguments:

Parameters
<outfname>(char*) name of the output file
<L>(int) length of the chain
<D>(int) maximum bond dimension
<g>(double) parameter $g$ of the Hamiltonian
<gamma>(double) parameter $gamma$ of the dissipation
<scal>(double) scale factor to multiply Lmpo
<tol>(double) tolerance asked from Contractor
<dir>(char*) directory where the output is to be written
<oldfile>[OPTIONAL] (char*) name of another (binary) file, containing an earlier MPS to be used as starting point

Program coherentAdiab: Compute the steady state of a certain Liouvillian as in with g, gamma given as input parameters. In this version, another MPS computed for the same D and slightly larger g is used as input, and when the program finishes, it starts a new job for a smaller g.

To compile: make coherentAd

Receives as argument a Properties file, such as config/excitProp.conf, from where the required parameters are read. Additional parameters can be given to override the values in the Properties file. To replace some of the values in the file by a command line option, the additional arguments have to be of the form -propName=propValue

The properties that will be required:

  • L (int) length of the chain
  • D (int) maximum bond dimension
  • g (double) parameter $g$ of the Hamiltonian
  • deltaG (double) change in the parameter $g$ of the Hamiltonian for the next job
  • gamma (double) parameter $gamma$ of the dissipation
  • scale (double) scale factor to multiply Lmpo
  • tol (double) tolerance asked from Contractor
  • mpsfile (string) file where the resulting MPS will be stored.
  • oldmpsfile (string) name of another (binary) file, containing an earlier MPS to be used as starting point
  • outdir (string) directory where to write the (automatically named) file(s) containing observables.

Program coherentPropsRand: Compute the steady state of a certain Liouvillian as in, gamma given as input parameters, starting from a random state (do not worry about positivity). An index is received to label the random attempt, with the objective of later comparing different runs, and extracting a number of linearly independent states.

To compile: make coherentRand

Receives as argument a Properties file, such as config/excitProp.conf, from where the required parameters are read. Additional parameters can be given to override the values in the Properties file. To replace some of the values in the file by a command line option, the additional arguments have to be of the form -propName=propValue

The properties that will be required:

  • L (int) length of the chain
  • D (int) maximum bond dimension
  • g (double) parameter $g$ of the Hamiltonian
  • gamma (double) parameter $gamma$ of the dissipation
  • scale (double) scale factor to multiply Lmpo
  • tol (double) tolerance asked from Contractor
  • randId (int) numbre of the random attempt, which will be added to file names
  • mpsdir (string) directory where the resulting MPS will be stored.
  • outdir (string) directory where to write the (automatically named) file(s) containing observables.
  • freqSv (int) if present, intermediate results of the findGrounState calculation are saved, every freqSv sweeps.
  • sym (bool) if not zero, the initial random state is constructed with reflection symmetry.
